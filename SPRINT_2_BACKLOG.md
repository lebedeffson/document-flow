# Sprint 2 Backlog

## Цель спринта

Сделать интеграцию между `Rukovoditel`, `middleware` и `NauDoc` менее ручной и более заметной для пользователей.

К концу спринта система должна:

1. лучше показывать состояние документа в рабочих карточках
2. синхронизировать связи стабильнее
3. поддерживать не только заявки, но и проектный сценарий

## Что уже сделано в старте Sprint 2

1. Добавлена связь проекта с карточкой документа.
2. Реализован сценарий `project -> document card -> middleware`.
3. Обратная синхронизация умеет подтягивать проектные bridge-записи обратно в `Rukovoditel`.
4. Добавлен background sync worker.
5. В проектах и заявках появился видимый статус `Статус документа / интеграции`.
6. В `middleware` появились counters, quick filters и журнал незавершенных связей.
7. В `Rukovoditel` добавлены отчеты по сущностям без финального документа.

## Границы Sprint 2

В этом спринте делаем:

1. второй интеграционный сценарий `проект -> карточка документа`
2. автозапуск обратной синхронизации
3. статус интеграции в карточках и списках
4. базовый журнал ошибок и проблемных связей

В этом спринте не делаем:

1. полную автоматическую генерацию документов в `NauDoc`
2. полноценный `SSO`
3. финальный продовый контур
4. большую переписку legacy

## Основные задачи

### 1. Интеграционный сценарий проекта

1. Создать сценарий `project -> document card`.
2. Привязать карточку документа к проекту.
3. Публиковать bridge-запись в `middleware`.
4. Подтягивать обратно финальную ссылку и статус.

## 2. Автоматизация синхронизации

1. Подготовить периодический запуск `pull_bridge_updates.sh`.
2. Сделать безопасный режим повторного запуска без дублей.
3. Добавить фильтр по измененным связям.
4. Подготовить лог выполнения синхронизации.

## 3. Видимость интеграции в интерфейсе

1. Показать статус связанного документа в карточке заявки.
2. Показать статус связанного документа в карточке проекта.
3. Показать статус интеграции в списках.
4. Выделить проблемные или незавершенные связи.

## 4. Контроль ошибок

1. Добавить список связей с ошибкой синхронизации.
2. Добавить понятные статусы:
   - ожидает документ
   - связано
   - подписан
   - ошибка синхронизации
3. Добавить повторную обработку проблемной связи.

## 5. Управленческая полезность

1. Добавить отчет:
   - документы без финальной ссылки `NauDoc`
2. Добавить отчет:
   - документы на согласовании
3. Добавить отчет:
   - проекты с документами без завершения маршрута

## Definition of Done

Спринт считается выполненным, если:

1. проект можно связать с карточкой документа так же, как уже связана заявка
2. обратная синхронизация может запускаться автоматически
3. пользователь видит состояние документа в карточке без ручного поиска
4. проблемные связи можно быстро найти и переобработать
5. изменения воспроизводимы скриптами

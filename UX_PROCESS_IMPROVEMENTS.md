# Улучшение UX и процессов

Дата фиксации: `2026-03-24`

## Зачем это делаем

Базовый контур уже стабилен:

1. `NauDoc` держит документный слой
2. `Rukovoditel` стал рабочим интерфейсом
3. `middleware` синхронизирует связки и статусы

Теперь главная задача не “еще раз оживить legacy”, а сделать систему понятнее для:

1. сотрудников
2. руководителей / преподавателей
3. канцелярии

## Что уже улучшено

### В интерфейсе

1. обновлен визуальный слой `Rukovoditel`
2. собраны ролевые меню
3. очищены формы от части служебных полей
4. добавлены базовые управленческие отчеты

### В процессах

1. есть сущности для проектов, заявок, документов и МТЗ
2. есть жизненный цикл карточек документов
3. есть мост к `NauDoc`
4. статусы документа уже возвращаются в рабочий контур

## Что нужно улучшать следующим пакетом

### 1. Стартовые рабочие экраны

Для каждой роли нужен свой понятный первый экран.

Сотрудник:

1. мои заявки
2. мои задачи
3. документы, где нужно действие

Руководитель:

1. просроченные задачи
2. документы на согласовании
3. проекты с риском
4. заявки без финального документа

Канцелярия:

1. документы на регистрации
2. документы без регистрационного номера
3. документы, ожидающие архивации

### 2. Карточки сущностей

Карточки должны читаться сверху вниз как рабочий сценарий.

Нужно привести к общей структуре:

1. кто инициатор
2. что требуется
3. кто отвечает
4. срок
5. статус
6. связанный документ
7. результат

### 3. Навигация по документному контуру

Пользователь не должен гадать, куда идти дальше.

Нужно явно подсветить:

1. есть ли карточка документа
2. есть ли ссылка на `NauDoc`
3. какой сейчас документный статус
4. требуется ли согласование / регистрация / подписание

### 4. Каналы входа заявок

Чтобы система была удобной в жизни, нужны легкие точки входа:

1. web-форма
2. email -> заявка
3. позже: мессенджер -> заявка

### 5. Руководительский контроль

Нужны короткие и понятные показатели:

1. просрочено
2. в работе
3. без исполнителя
4. без финального документа
5. ожидает согласования

### 6. Канцелярский сценарий

Нужно довести маршрут:

1. создана карточка
2. документ направлен в `NauDoc`
3. регистрация
4. согласование
5. подпись
6. ознакомление
7. архив

## Что можно делать на текущем стеке

Без переписывания ядра уже можно:

1. улучшать карточки и списки в `Rukovoditel`
2. усиливать дашборды
3. делать role-based UX
4. добавлять входные формы заявок
5. упрощать путь до документа в `NauDoc`

## Что не стоит делать прямо сейчас

Пока не стоит:

1. переписывать `NauDoc` целиком
2. переносить документное ядро без пилота
3. ломать старые маршруты ради красивого фронта

## Ближайший практический пакет

Я бы вел развитие так:

1. дожать UX карточек и списков
2. усилить руководительские панели
3. сделать входную форму заявки
4. добавить инструкции по ролям
5. после этого идти в пилот

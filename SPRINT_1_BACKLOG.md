# Sprint 1 Backlog

## Цель спринта

Превратить текущий стенд в понятный и управляемый рабочий каркас системы:

1. с четкими ролями
2. с едиными статусами
3. с удобными формами
4. с первыми управленческими представлениями

## Границы Sprint 1

В этом спринте делаем:

1. роли и базовые права доступа
2. словарь статусов по ключевым сущностям
3. доводку форм и списков
4. связи между рабочими сущностями и документами
5. первые общие отчеты для дашбордов

В этом спринте не делаем:

1. полную синхронизацию с `NauDoc`
2. полноценный workflow-движок в `middleware`
3. продовую инфраструктуру
4. email-автоматизацию и ботов

## Основные задачи

### 1. Роли и доступ

1. Зафиксировать роли:
   - `Руководитель / преподаватель`
   - `Исполнитель / сотрудник`
   - `Заявитель / пользователь`
2. Проверить права по ключевым сущностям:
   - проекты
   - задачи
   - заявки на обслуживание
   - карточки документов
   - база документов
   - заявки на МТЗ

### 2. Формы и обязательные поля

1. Довести форму заявок на обслуживание.
2. Добавить приоритет заявок.
3. Добавить результат исполнения заявки.
4. Довести форму заявок на МТЗ.
5. Добавить инициатора / группу для МТЗ.
6. Сделать связи с проектами и карточками документов более заметными в списках.

### 3. Словарь статусов

1. Проекты:
   - инициирование
   - в работе
   - на согласовании
   - на паузе
   - завершен
   - в архиве
2. Задачи:
   - новая
   - в работе
   - на согласовании
   - на проверке
   - выполнена
   - отменена
3. Заявки:
   - новая
   - принята
   - в работе
   - на согласовании
   - ожидает заявителя
   - выполнена
   - отклонена
4. Документы:
   - черновик
   - на согласовании
   - на утверждении
   - подписан
   - на ознакомлении
   - архивирован
5. МТЗ:
   - новая
   - на согласовании
   - в закупке
   - заказано
   - получено
   - отклонена

### 4. Первые дашборды

1. Для сотрудников:
   - `Мои задачи в работе`
   - `Мои заявки`
2. Для руководителей:
   - `Проекты на контроле`
   - `Документы на согласовании`
   - `Заявки МТЗ в работе`

## Definition of Done

Спринт считается выполненным, если:

1. роли и права доступа зафиксированы в системе
2. формы не перегружены и содержат ключевые поля
3. приоритеты и статусы единообразны
4. в списках видны связанные проектные и документные сущности
5. в системе созданы общие отчеты под будущие дашборды
6. изменения воспроизводимы через provisioning-скрипт

# Пользовательский гайд по входу и проверке платформы

Дата актуальности: `2026-03-27`

## 1. Основные адреса

Использовать только:

1. Платформа: `https://localhost:18443/`
2. NauDoc: `https://localhost:18443/docs/`
3. Bridge: `https://localhost:18443/bridge/`

Не использовать:

1. `https://localhost/docs/`
2. прямые внутренние порты
3. прямой URL `ONLYOFFICE`

## 2. Что реально встроено

Уже работает:

1. `Rukovoditel`
2. `NauDoc`
3. `ONLYOFFICE Docs`
4. `Bridge`

Пока не развернуты как отдельные сервисы:

1. `ONLYOFFICE DocSpace`
2. `ONLYOFFICE Workspace`

## 3. Два режима платформы

### Админский режим

Вход:

- `admin / admin123`

Нужен для:

1. настройки платформы
2. проверки интеграции
3. контроля логов
4. проверки резервных копий
5. полного обзора всех модулей

### Пользовательский режим

Вход:

- `user.demo / rolepass123`

Нужен для:

1. проверки чистого рабочего интерфейса
2. проверки модулей без админского шума
3. проверки обычного маршрута пользователя

## 4. Куда идти под пользователем

После входа под `user.demo` проверять:

1. `Заявки на обслуживание`
2. `Поручения и задачи`
3. `Проекты и инициативы`
4. `Карточки документов`
5. `База документов`
6. `Заявки на МТЗ`

Главная под пользователем должна быть простой и без служебных разделов.

## 5. Куда идти под админом

После входа под `admin` проверять:

1. главную администратора
2. пользователей и роли
3. настройки приложения
4. сущности приложения
5. группы доступа
6. резервные копии
7. логи
8. Bridge
9. рабочие модули

## 6. Как проверять документ

Правильный путь:

1. открыть `Карточки документов`
2. открыть карточку документа
3. нажать `Открыть документ в редакторе`
4. при необходимости открыть `Открыть в NauDoc`

## 7. Как проверять NauDoc

Открыть:

- `https://localhost:18443/docs/`
- логин: `admin`
- пароль: смотреть `NAUDOC_PASSWORD` в корневом `.env`

Проверить:

1. главную страницу
2. хранилище
3. справочники
4. системные страницы документа

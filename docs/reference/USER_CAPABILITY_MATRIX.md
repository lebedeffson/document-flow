# User Capability Matrix

Дата: `2026-03-28`

## Назначение

Этот документ фиксирует, что реально может делать обычный пользователь платформы без административных полномочий.

## Учетная запись для теста

1. логин: рабочая учетная запись сотрудника
2. пароль: из `DOCFLOW_ROLE_DEFAULT_PASSWORD`

## Что пользователь видит в контуре

1. главную страницу рабочего кабинета
2. заявки
3. задачи
4. проекты
5. карточки документов
6. базу документов
7. МТЗ
8. встроенный переход в `ONLYOFFICE`
9. переход в `NauDoc`

## Что пользователь реально может делать

### 1. Работать с заявками

1. открыть список заявок
2. создать заявку
3. открыть карточку заявки
4. просматривать связанный документный контур

### 2. Работать с задачами

1. открыть отчет по задачам
2. просматривать свои поручения
3. переходить в связанные проекты и документы

### 3. Работать с проектами

1. открыть список проектов
2. просматривать проект
3. переходить в связанные документы

### 4. Работать с карточками документов

1. открыть список карточек документов
2. искать документы через строку поиска
3. открыть карточку документа
4. создать новую карточку документа
5. загрузить вложение
6. открыть документ в `ONLYOFFICE`
7. открыть связанный объект в `NauDoc`

### 5. Работать с базой документов

1. открыть список шаблонов и материалов
2. искать нужный шаблон
3. открыть карточку шаблона
4. использовать шаблон как основу для работы

### 6. Работать с МТЗ

1. открыть список заявок МТЗ
2. открыть карточку заявки
3. просматривать связь с проектом и документами

## Что уже проверено

1. карточки документов `#14-#17` открываются
2. шаблоны `#6-#9` открываются
3. `ONLYOFFICE` открывается под рабочей учетной записью сотрудника
4. документы получают конкретные ссылки в `NauDoc`
5. пользовательский `sidebar` проходит browser audit без ошибок

## Что пользователь не должен делать

1. не работает с `Bridge`
2. не управляет пользователями и ролями
3. не меняет маппинги и связи интеграции
4. не работает с логами, backup и системными настройками

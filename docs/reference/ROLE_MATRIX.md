# Матрица ролей и прав

Дата: `2026-03-24`

## 1. Назначение

Этот документ фиксирует рабочую матрицу ролей для production-lite контура:

`Rukovoditel + middleware + NauDoc`

Документ нужен для:

1. настройки прав в системе
2. назначения пользователей по группам
3. пилота на реальных подразделениях
4. последующей интеграции с `SSO/LDAP`

## 2. Роли

### `admin`

- системный администратор
- полные права на все сущности
- отвечает за конфигурацию, интеграцию, восстановление и сопровождение

### `4` Заведующий отделением / руководитель подразделения

- согласование и утверждение
- управленческий контроль
- просмотр проектов, заявок, документов и МТЗ
- доступ к руководительским отчетам

### `5` Врач / сотрудник подразделения

- ежедневная работа в системе
- ведение проектов, задач, заявок и карточек документов
- исполнение поручений
- подготовка материалов и вложений

### `6` Регистратура / заявитель

- подача заявок
- отслеживание собственных обращений
- ограниченный доступ к связанным записям

### `7` Канцелярия / делопроизводство

- регистрация документов
- контроль карточек и маршрутов
- сопровождение архива
- чтение связанных проектов и заявок

## 3. Словарь прав

- `view` - видеть все записи сущности
- `view_assigned` - видеть только связанные / назначенные записи
- `create` - создавать записи
- `update` - редактировать записи
- `delete` - удалять записи
- `reports` - видеть отчеты и панели по сущности

## 5. Практический смысл ролей

### Сотрудник / исполнитель

Работает в основном в:

1. заявках
2. задачах
3. проектах
4. карточках документов

Не должен:

1. управлять системными настройками
2. видеть все чужие обсуждения без связи с задачей
3. заменять канцелярский контур регистрации

### Заведующий отделением / руководитель подразделения

Работает в основном как:

1. согласующий
2. утверждающий
3. контролер сроков и нагрузки
4. владелец отчетов и панелей

### Заявитель

Работает по принципу:

1. подал заявку
2. видит свой статус
3. может дать уточнение
4. не получает избыточный доступ к чужим данным

### Канцелярия

Работает в:

1. карточках документов
2. базе документов
3. контрольных отчетах по регистрации и архиву

Имеет read-only контекст по:

1. проектам
2. задачам
3. сервисным заявкам

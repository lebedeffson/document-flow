# Pilot Runbook

## Purpose
Этот runbook фиксирует минимально безопасный сценарий пилота платформы на одном подразделении больницы.

## Recommended pilot contour
1. Одно подразделение.
2. Один ответственный администратор.
3. Один руководитель подразделения.
4. 3-10 сотрудников-пользователей.
5. Один сотрудник канцелярии или регистратуры.

## Entry points
1. Платформа: `https://<hospital-domain>/`
2. NauDoc: `https://<hospital-domain>/docs/`
3. Bridge: `https://<hospital-domain>/bridge/`

## Pilot goals
1. Проверить реальный цикл создания документа.
2. Проверить редактирование через ONLYOFFICE.
3. Проверить публикацию и официальный контур NauDoc.
4. Проверить роли и доступы.
5. Проверить журнал sync-ошибок и retry.

## Minimal scope for wave 1
1. Входящий документ.
2. Внутренний документ подразделения.
3. Один шаблон документа.
4. Один маршрут согласования.

## Roles for pilot
1. `Администратор платформы`
2. `Заведующий отделением`
3. `Врач / сотрудник`
4. `Регистратура`
5. `Канцелярия`

## Acceptance checklist
1. Пользователь входит под своей учетной записью.
2. Создает карточку документа.
3. Загружает вложение.
4. Открывает документ в ONLYOFFICE.
5. Сохраняет изменения.
6. Документ получает конкретный объект в NauDoc.
7. Карточка и NauDoc-ссылка видны в системе.
8. Администратор видит связь и health через Bridge.

## Operational checklist before pilot
1. Боевой `.env` активирован.
2. Backup и restore drill пройдены.
3. `check_stack.sh` проходит.
4. `run_full_verification.sh` проходит.
5. LDAP/каталог пользователей синхронизирован.
6. Доступы и роли подтверждены.

## Success criteria
1. Пользователи работают без ручной правки БД.
2. Документы редактируются и находятся через GUI.
3. Интеграция не требует ручного вмешательства в обычном сценарии.
4. Ошибки синхронизации, если возникают, видны и повторно запускаются через Bridge.

## Exit criteria from pilot
1. Подтвержден стабильный рабочий маршрут.
2. Подтверждены роли и видимость документов.
3. Подтверждено восстановление из backup.
4. Сформирован список улучшений для полного rollout.

Ниже готовый стартовый набор пользователей для первого пилота на одном подразделении.

Рекомендуемый сценарий:
1. администратор платформы заводит этих людей через `Пользователи -> Создать пользователя`
2. каждому назначается нужная группа доступа
3. реальные пароли задаются отдельно на стороне учреждения

## Минимальный набор для первого запуска

Если нужно стартовать совсем быстро, достаточно 4 ролей:
1. `Администратор платформы`
2. `Заведующий отделением`
3. `Врач отделения`
4. `Канцелярия`

## Что важно не забыть при заведении

1. Указать корректную группу доступа.
2. Проверить email и логин.
3. Задать реальные пароли отдельно от demo-стенда.
4. Если используется LDAP/AD, логины должны совпадать с корпоративным каталогом.
5. Для пилота лучше не смешивать тестовых и реальных людей в одном подразделении.

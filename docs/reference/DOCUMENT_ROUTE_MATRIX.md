# Матрица маршрутов документов

Дата: `2026-03-24`

## 1. Назначение

Этот документ фиксирует типовые hospital-маршруты документов для production-lite контура.

Цель:

1. синхронизировать бизнес-логику между `Rukovoditel`, `middleware` и `NauDoc`
2. зафиксировать единый словарь статусов
3. убрать разночтения при настройке карточек и процессов

## 2. Единый словарь статусов документа

| Статус | Смысл | Основной владелец шага |
|---|---|---|
| `Черновик` | карточка создана, документ готовится | инициатор / исполнитель |
| `На согласовании` | документ проходит согласование по маршруту | согласующие |
| `На утверждении` | документ передан руководителю на финальное утверждение | заведующий / руководитель подразделения |
| `Подписан` | документ утвержден и подписан | подписант |
| `На ознакомлении` | документ доводится до исполнителей или сотрудников | руководитель подразделения / канцелярия |
| `На регистрации` | документ передан в канцелярский контур для учета | канцелярия |
| `Зарегистрирован` | документ имеет регистрационный номер и считается оформленным | канцелярия |
| `Архивирован` | документ завершил жизненный цикл и переведен в архив | канцелярия / администратор |

## 3. Интеграционные статусы для проектов и заявок

Эти статусы нужны в `Rukovoditel` для полей `Статус документа / интеграции`.

| Статус | Смысл |
|---|---|
| `Ожидает документ` | документ еще не создан или не связан |
| `Связано` | карточка связана, но формальный маршрут еще не начался |
| `Черновик` | документ создан и находится в работе |
| `На согласовании` | документ согласуется |
| `На утверждении` | документ на финальном утверждении |
| `Подписан` | документ подписан |
| `На ознакомлении` | документ доводится до пользователей |
| `Зарегистрирован` | документ зарегистрирован |
| `Архивирован` | документ завершен и архивирован |
| `Ошибка синхронизации` | интеграция не завершилась корректно |

## 4. Где находится “истина”

| Контур | Что является мастер-данными |
|---|---|
| `Rukovoditel` | проекты, заявки, задачи, рабочие карточки |
| `middleware` | связь сущностей, технический статус интеграции, журналы связи |
| `NauDoc` | документ, версия, согласование, подпись, архив |

## 5. Типовые маршруты

## 5.1 Входящий документ

### Основной сценарий

1. `Черновик`
2. `На регистрации`
3. `Зарегистрирован`
4. `На ознакомлении`
5. `Архивирован`

### Обязательные поля

1. наименование документа
2. тип документа
3. дата документа
4. регистрационный номер
5. ответственный
6. срок исполнения или срок действия

### Кто инициирует

1. канцелярия
2. ответственный сотрудник по входящему документу

## 7. Роли по этапам маршрута

| Этап | Кто делает |
|---|---|
| Создание карточки | врач / сотрудник / канцелярия |
| Согласование | согласующие по маршруту |
| Утверждение | заведующий / руководитель подразделения / уполномоченный |
| Подписание | подписант |
| Ознакомление | руководитель подразделения / канцелярия |
| Регистрация | канцелярия |
| Архив | канцелярия / администратор |

## 9. Что нужно сделать следующим шагом

1. привязать маршруты к конкретным категориям документов
2. настроить обязательные поля по типу документа
3. завести правила переходов статусов в `middleware`
4. синхронизировать статусы `NauDoc -> Rukovoditel`
5. выделить hospital-маршруты:
   - входящая регистрация
   - внутренний приказ
   - медицинский документ
   - маршрут пациента / направления
   - договор / закупка

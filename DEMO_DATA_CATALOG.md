# Demo Data Catalog

Этот каталог фиксирует осмысленные примеры по основным модулям и отчетам тестового контура.

## Как обновить demo-данные

```bash
cd /home/lebedeffson/Code/Документооборот/rukovoditel-test
bash prepare_customer_demo.sh
```

## Проекты

- `Внедрение единой платформы документооборота`
  - основной сквозной кейс: проект -> карточка документа -> NauDoc -> ONLYOFFICE
- `Подготовка регламента согласования учебных документов`
  - кейс с незавершенным документным циклом и риском по документам
- `Архивирование завершенных регламентов`
  - завершенный кейс для показа полного жизненного цикла

## Поручения и задачи

- `Подготовить текст приказа о запуске пилота`
  - пример активной задачи по подготовке документа
- `Проверить регламент согласования учебных документов`
  - пример задачи в согласовании
- `Собрать замечания канцелярии к входящему договору`
  - пример межрольного взаимодействия
- `Подготовить комплект закупки МФУ для канцелярии`
  - пример завершенной задачи

## Заявки на обслуживание

- `Подготовка маршрута согласования приказа о пилоте`
  - основной кейс сервисной заявки, уже связанной с документом
- `Подключить шаблон служебной записки для кафедр`
  - заявка в работе без финальной версии документа
- `Закупка принтера для регистратора канцелярии`
  - пример хозяйственной заявки
- `Оформить карточку входящего договора на сопровождение`
  - пример канцелярского запроса с документом
- `Открыть доступ к шаблонам служебной записки`
  - пример пользовательской заявки от врача подразделения

## Ключевой demo-маршрут

1. Открыть `Внедрение единой платформы документооборота`.
2. Перейти в связанную карточку документа `Документ проекта: Внедрение единой платформы документооборота`.
3. Открыть `Служебная записка о запуске пилотного контура`.
4. Нажать `Открыть документ в редакторе`.
5. Вернуться и открыть `NauDoc`.
6. Показать отчеты `Проекты на контроле`, `Документы на согласовании`, `Заявки без финального документа`.
